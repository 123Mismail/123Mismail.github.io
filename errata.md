# Errata: Physical AI & Humanoid Robotics Textbook

**Purpose**: Track critical errors discovered after chapter approval that require emergency corrections.

**Correction Policy** (per Constitution v1.1.0 - Emergency Post-Approval Corrections):
- Only critical errors (broken code, mathematical errors, incorrect API references) can be fixed post-approval
- Non-critical improvements (style, wording) must wait for next edition
- Each correction must document: Chapter, Error Description, Fix Applied, Date

## Format

```
### [Chapter ID] - [Error Type]
**Date Discovered**: YYYY-MM-DD
**Severity**: Critical | High | Medium
**Error**: [Description of the error]
**Fix Applied**: [Description of the correction]
**Verified**: [Date fix was verified]
```
